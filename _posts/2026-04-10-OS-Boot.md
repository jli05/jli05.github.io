This is a companion to the [Booting the kernel](https://operating-system-in-1000-lines.vercel.app/en/04-boot) in RISC-V32.

Note the RISC-V stack is 16-byte aligned, so the `kernel.ld` need be modified.

# The C code
Add `-mcmodel=medany` for medium any [memory code model](https://www.sifive.com/blog/all-aboard-part-4-risc-v-code-models).

```sh
clang -std=c11 --target=riscv64-unknown-elf -c -ffreestanding -nostdlib -mcmodel=medany -o kernel.o kernel.c
ld.lld -Tkernel.ld -o kernel.elf kernel.o
```

# A simplest kernel in assembly
Let's write a `kernel2.s` that only consists of a `boot()` function, which only sets the stack pointer `sp`. Remove `.data`, `.rodata`, `.bss` from the linker file.

```asm
.global boot

.extern __stack_top

.section .text.boot
boot:
    la sp, __stack_top
```

Note the memory code model parameter `-mcmodel` is no longer needed.

```sh
clang --target=riscv64-unknown-elf -c -nostdlib -o kernel2.o kernel2.s
ld.lld -Tkernel2.ld -o kernel2.elf kernel2.o
```
